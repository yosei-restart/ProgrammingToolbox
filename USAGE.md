## 一、快速入门

### 1.1 系统要求

| 项目 | 最低要求 | 推荐 |
|------|---------|------|
| 操作系统 | Windows 10 1903+ | Windows 11 |
| Python（源码运行） | 3.10 | 3.11+ |
| 内存 | 2 GB | 8 GB+ |
| 磁盘空间 | 50 MB（源码）/ 250 MB（EXE） | 1 GB |
| 屏幕分辨率 | 1366×768 | 1920×1080+ |

### 1.2 安装方式

#### 方式一：EXE 直接运行（推荐）

1. 前往 [Releases](https://github.com/yosei-restart/ProgrammingToolbox/releases) 页面
2. 下载最新版 `ProgrammingToolbox.exe`
3. 双击运行，无需安装

#### 方式二：源码运行

```bash
# 克隆仓库
git clone https://github.com/yosei-restart/ProgrammingToolbox.git
cd programming-toolbox

# 创建虚拟环境
python -m venv venv
venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动
python main.py
```

或直接运行启动脚本：

```
双击 启动.bat
```

### 1.3 首次启动

启动后，你会看到工具箱主界面，包含 **8 个工具卡片**。每个卡片显示工具名称、简介和一个彩色图标。

> 💡 **提示**：首次使用时，状态栏会显示当前拾取热键（默认 `Ctrl+F2`）。如需修改，请参阅 [热键配置](#十二热键配置) 章节。

## 三、工具一：GUI 元素探查器

### 3.1 功能简介

点击任意 GUI 控件，自动识别其类型、属性、父级层级，并生成带高亮标注的截图和 AI 提示词。

### 3.2 使用步骤

#### 步骤 1：进入检查模式

- **方式一**：按拾取热键（默认 `Ctrl+F2`）
- **方式二**：点击工具箱中的"GUI 元素探查器"卡片，再点击主窗口的"拾取"按钮

进入检查模式后，光标会变为**十字准星**。

#### 步骤 2：点击目标控件

将十字准星移动到目标控件上，**左键或右键点击**均可触发识别。

#### 步骤 3：查看识别结果

识别完成后，主窗口右侧详情面板会显示以下分区：

| 分区 | 说明 |
|------|------|
| 功能引导 | 控件类型说明和操作建议 |
| 控件作用 | 控件类型的中文名称和功能描述 |
| 推测框架 | UI 框架推测结果及置信度 |
| 控件属性 | 完整属性列表（类型、类名、名称、AutomationId、位置、尺寸、状态等） |
| 控件层级 | 从当前控件到顶层窗口的父级链 |
| 截图标注 | 全屏高亮截图 + 控件特写截图 |
| AI 提示词 | 可直接粘贴到 AI 对话的结构化文本 |

#### 步骤 4：复制或导出

- **复制单个属性**：点击属性右侧的"复制"按钮
- **复制全部属性**：点击"复制全部"按钮
- **复制 AI 提示词**：在"AI 提示词"区编辑你的问题，点击"复制提示词"
- **导出 JSON**：点击"导出 JSON"按钮，保存结构化数据

### 3.3 拾取多个控件

- 每次拾取后，控件会自动添加到左侧"已拾取列表"
- 点击列表项可切换查看不同控件的详情
- 点击"清空"按钮清除所有拾取记录

### 3.4 识别的属性

| 属性 | 说明 | 示例 |
|------|------|------|
| ControlType | 控件类型 | Button |
| ClassName | 类名 | Button |
| Name | 控件名称 | 确定 |
| AutomationId | 自动化 ID | btn_ok |
| BoundingRectangle | 位置和尺寸 | (100, 200, 80, 32) |
| FrameworkId | 框架标识 | Win32 |
| ProcessName | 所属进程 | notepad.exe |
| Handle | 窗口句柄 | 0x00012345 |
| SupportedPatterns | 支持的交互模式 | Invoke, Value |
| Value | 当前值 | 确定 |

### 3.5 支持的控件类型

内置 **37 种**控件类型的中文描述，包括：

- 按钮类：Button、RadioButton、CheckBox、SplitButton
- 文本类：Edit、Text、Hyperlink
- 选择类：ComboBox、ListBox、Slider
- 容器类：Pane、Window、Dialog、Group
- 树形类：Tree、TreeItem、DataGrid、DataItem
- 菜单类：Menu、MenuItem、MenuBar
- 其他类：Tab、TabItem、ToolTip、StatusBar、ProgressBar、Spinner、Thumb

## 四、工具二：变量生命周期追踪器（静态）

### 4.1 功能简介

基于 AST（抽象语法树）分析 Python 源码，追踪变量从诞生到消亡的完整生命周期。

### 4.2 使用步骤

1. 点击工具箱中的"变量生命周期追踪器（静态）"卡片
2. 在打开的窗口中，选择目标 Python 文件或文件夹
3. 输入要追踪的变量名
4. 点击"开始分析"
5. 查看分析结果

### 4.3 变量生命周期四阶段

| 阶段 | 图标 | 说明 | 示例 |
|------|------|------|------|
| 诞生 | 🟢 | 变量首次定义 | `x = 10` |
| 赋值 | 🟡 | 变量值发生变化 | `x = 20` |
| 使用 | 🔵 | 变量被读取或传参 | `print(x)` |
| 销毁 | 🔴 | 变量作用域结束 | 函数返回 |

### 4.4 导出报告

- 点击"导出 HTML"按钮
- 生成的 HTML 报告包含：
  - 变量生命周期时间线
  - 每个事件的代码片段（含行号）
  - 可点击跳转到源码位置

## 五、工具三：变量生命周期追踪器（动态）

### 5.1 功能简介

运行 Python 程序时实时追踪变量值的变化，记录运行时类型和真实数据。

### 5.2 使用步骤

1. 点击工具箱中的"变量生命周期追踪器（动态）"卡片
2. 选择目标 Python 脚本
3. 输入要追踪的变量名
4. 点击"开始追踪"
5. 程序运行时，实时查看变量值变化
6. 追踪完成或点击"强制停止"结束

### 5.3 关键特性

- **预计算目标文件**：使用 O(1) 集合查找，仅追踪包含目标变量的文件
- **call 事件过滤**：仅在包含目标变量的函数上安装追踪器
- **延迟一行机制**（Delay-One-Line）：确保诞生/赋值事件行号与静态分析一致
- **强制停止按钮**：标题栏右侧红色按钮，调用 `QProcess.kill()` 终止目标程序

### 5.4 注意事项

- ⚠️ 追踪会影响程序运行性能，建议仅在小范围使用
- ⚠️ 子进程退出依赖 `call_count` 检测（连续 15 秒不变化判定为停止）
- ⚠️ 关闭追踪窗口时会先 kill QProcess，避免子进程成为孤儿

## 六、工具四：函数调用链分析器

### 6.1 功能简介

基于 AST 分析 Python 源码，构建函数调用关系图，支持查看调用链和被调用方。

### 6.2 使用步骤

1. 点击工具箱中的"函数调用链分析器"卡片
2. 选择目标 Python 文件或文件夹
3. 点击"开始分析"
4. 在函数列表中选择目标函数
5. 查看：
   - **调用链**：该函数调用了哪些其他函数
   - **被调用方**：哪些函数调用了该函数

## 九、工具七：ML 模型选择器

### 9.1 功能简介

通过渐进式问答，根据数据特征和使用场景，推荐最适合的机器学习模型。

### 9.2 使用步骤

1. 点击工具箱中的"ML 模型选择器"卡片
2. 回答一系列问题：
   - 任务类型（分类/回归/聚类/降维/异常检测/时序预测）
   - 数据规模
   - 样本数量
   - 异常样本数量（如适用）
   - 精度要求
   - 训练时间要求
3. 查看推荐结果

### 9.3 判断标准（量化）

#### 分类 vs 异常检测

| 条件 | 推荐方向 |
|------|---------|
| 异常样本 ≥ 20 个 **AND** 占比 ≥ 2% | 分类任务 |
| 异常样本 < 20 个 **OR** 占比 < 2% | 异常检测任务 |

#### 数据规模判断

| 数据量 | 规模 | 适用模型 |
|--------|------|---------|
| < 5,000 | 小 | 轻量模型（如 Logistic Regression） |
| 5,000 - 100,000 | 中 | 标准模型（如 Random Forest） |
| > 100,000 | 大 | 深度学习模型 |

## 十、工具八：快速截图标注

### 10.1 功能简介

通过全局热键触发全屏截图，提供红框、箭头、文本框三种标注工具，支持一键复制或保存。

### 10.2 使用步骤

1. **触发截图**：按截图热键（可在设置中配置）
2. **选择标注工具**：
   - 🔲 **红框**：按住鼠标拖动绘制矩形框
   - ➡️ **箭头**：按住鼠标拖动绘制箭头
   - 📝 **文本框**：点击放置文本框，输入文字
3. **调整文本框**：
   - 拖动四角缩放手柄调整大小
   - 拖动文本框内部移动位置
4. **确认截图**：
   - 📋 **复制到剪贴板**：点击"复制"按钮
   - 💾 **另存为**：点击"保存"按钮
5. **退出**：点击"✕"按钮或按 `ESC` 键

### 10.3 标注规范

| 标注类型 | 颜色 | 绘制方式 |
|---------|------|---------|
| 红框 | 纯红 RGB(255,0,0) | 按住拖动，实时预览 |
| 箭头 | 纯红 RGB(255,0,0) | 按住拖动，实时预览 |
| 文本框 | 白底黑字 | 点击放置，双击编辑 |

### 10.4 快捷操作

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 取消当前绘制 | `ESC` | 取消未完成的标注（不退出截图模式） |
| 退出截图模式 | `ESC`（无绘制时） | 退出截图 |
| 关闭退出 | 点击 ✕ 按钮 | 触发完整 closeEvent 清理流程 |

### 10.5 注意事项

- 截图时**自动隐藏所有窗口**，确保截图干净
- 截图时**工具栏自动隐藏**，不截进画面
- 截图**自动过滤任务栏**，不包含任务栏区域
- 文本框默认大小为屏幕宽度的 1/5、高度的 1/10
- 确认截图后显示"已复制到剪贴板"提示（1 秒）并退出

## 十一、AI 分析功能

### 11.1 配置 AI 模型

#### 步骤 1：打开设置窗口

在工具箱主界面，点击**设置**按钮（或托盘菜单的"设置"）。

#### 步骤 2：配置文字语言模型

在"文字语言模型"标签页：

1. **选择网络模型**：从下拉框选择预设供应商
   - DeepSeek（默认 `deepseek-chat`）
   - OpenAI（默认 `gpt-4o-mini`）
   - 智谱 AI（默认 `glm-4-flash`，免费额度）
   - Agnes AI（默认 `agnes-2.0-flash`）
   - 本地 Ollama（默认 `qwen2.5`）
   - 本地 LM Studio（默认 `loaded-model`）
   - 自定义

2. **填写 Base URL**：选择预设后自动填充，也可手动修改
3. **填写 API Key**：粘贴你的密钥
4. **填写模型名称**：选择预设后自动填充，也可手动修改

#### 步骤 3：配置图像识别模型

切换到"图像识别模型"标签页，配置方式与文字模型相同。

> 💡 **切换 URL 自动切换模型名**：选择不同的 Base URL 时，模型名称会自动切换为对应默认值。

#### 步骤 4：测试连接

点击"测试连接"按钮：
- 在"文字语言模型"标签页：测试文字模型
- 在"图像识别模型"标签页：测试图像模型
- 在"AI 提示词"标签页：提示切换到文字/图像模型标签页测试

#### 步骤 5：保存配置

点击底部"保存配置"按钮，同时保存：
- 文字模型配置
- 图像模型配置
- 提示词配置

> 🔒 **安全说明**：API Key 存储在 `~/.gui_inspector/ai_config.json`，不在项目目录中，不会上传到 GitHub。

### 11.2 自定义提示词

#### 步骤 1：打开提示词配置

在设置窗口中，切换到"AI 提示词"标签页。

#### 步骤 2：编辑提示词

支持 4 种提示词模板：

| 提示词 | 键名 | 使用场景 |
|--------|------|---------|
| 控件分析 | `control_analysis` | 分析 GUI 控件的类型、作用、自动化方法 |
| 变量分析 | `variable_analysis` | 分析变量的生命周期和值变化 |
| 图片分析 | `image_analysis` | 分析截图中的界面内容 |
| 图片提示词反推 | `image_prompt_reverse` | 根据图片反推 AI 绘画提示词 |

#### 步骤 3：使用占位符

提示词中可使用以下占位符，系统会自动替换为实际数据：

| 占位符 | 替换内容 | 适用提示词 |
|--------|---------|-----------|
| `{control_info}` | 控件信息（类型、属性、层级等） | 控件分析 |
| `{variable_name}` | 变量名称 | 变量分析 |
| `{events_summary}` | 变量事件摘要 | 变量分析 |

**示例提示词**：

```
请分析以下 GUI 控件，并告诉我：
1. 这个控件的类型和作用
2. 如何用 Python 自动化操作它
3. 常见的交互模式

控件信息：
{control_info}
```

#### 步骤 4：保存提示词

点击底部"保存配置"按钮。

> 💡 **提示**：
> - 修改提示词后**立即生效**，无需重启程序
> - **删除全部内容**会自动恢复为默认值
> - 单项缺失时用默认值兜底，其他项保留用户自定义

### 11.3 AI 分析使用场景

#### 场景一：控件分析

1. 使用"GUI 元素探查器"拾取一个控件
2. 在详情面板点击"AI 分析"按钮
3. AI 会基于你的提示词和控件信息返回分析结果

#### 场景二：变量分析

1. 使用"变量生命周期追踪器"追踪变量
2. 在结果窗口点击"AI 分析"按钮
3. AI 会分析变量的生命周期和值变化

#### 场景三：图片分析

1. 使用"快速截图标注"截图
2. 点击"AI 图片分析"按钮
3. 截图工具会退出，AI 分析对话框作为独立窗口显示
4. AI 会分析截图中的界面内容

#### 场景四：图片提示词反推

1. 使用"快速截图标注"截图
2. 点击"AI 提示词分析"按钮
3. AI 会根据图片反推 AI 绘画提示词

> ⚠️ **注意**：点击"AI 图片分析"或"AI 提示词分析"后，截图工具会**彻底退出**，分析对话框作为独立窗口保留。分析完成后关闭对话框，主窗口会重新显示。

## 十二、热键配置

### 12.1 修改热键

#### 方式一：界面操作

1. 在主窗口点击"修改热键"按钮
2. 弹出热键捕获窗口
3. 按下新的热键组合（如 `Ctrl+Shift+I`）
4. 自动保存

#### 方式二：手动编辑配置

编辑项目目录下的 `config.json`：

```json
{
  "modifiers": ["Ctrl", "Shift"],
  "key": "i"
}
```

修改后重启程序生效。

### 12.2 热键规则

| 字段 | 类型 | 可选值 | 说明 |
|------|------|--------|------|
| `modifiers` | string[] | `Ctrl`、`Shift`、`Alt`、`Win` | 修饰键，可多选 |
| `key` | string | 字母或功能键名（小写） | 触发键 |

**示例**：

| 热键 | modifiers | key |
|------|-----------|-----|
| `Ctrl+F2` | `["Ctrl"]` | `"f2"` |
| `Ctrl+Shift+I` | `["Ctrl", "Shift"]` | `"i"` |
| `Alt+P` | `["Alt"]` | `"p"` |

### 12.3 ESC 键功能

在检查模式激活时，按 `ESC` 键可退出检查模式。

## 十四、日志与排查

### 14.1 日志位置

日志文件位于项目目录下的 `logs/` 文件夹：

```
logs/
└── inspector_YYYYMMDD.log
```

### 14.2 日志格式

```
YYYY-MM-DD HH:MM:SS [LEVEL] [module] [trace_id] message
```

**示例**：

```
2026-07-20 10:30:45 [INFO] [inspector_engine] [trc_20260720_abc12345] 控件识别成功: Button
```

### 14.3 日志级别

| 级别 | 说明 | 记录位置 |
|------|------|---------|
| DEBUG | 调试信息 | 文件 |
| INFO | 一般信息 | 文件 |
| WARNING | 警告 | 文件 + 控制台 |
| ERROR | 错误 | 文件 + 控制台 |
| CRITICAL | 严重错误 | 文件 + 控制台 |

### 14.4 trace_id

每个操作分配唯一的 `trace_id`，格式为 `trc_yyyymmdd_xxxxxxxx`，贯穿全链路，便于问题定位。

## 十六、常见问题（FAQ）

### Q1：拾取控件时无反应

**可能原因**：
1. 目标应用以管理员权限运行，本工具未以管理员权限运行
2. 控件未启用 UIAutomation
3. 热键冲突

**解决方案**：
1. 以管理员权限运行本工具
2. 尝试其他控件
3. 修改热键

### Q2：AI 分析超时

**可能原因**：
1. 网络不稳定
2. API 服务端响应慢
3. 提示词过长

**解决方案**：
1. 系统已设置 180 秒超时 + 2 次重试，请稍后再试
2. 检查网络连接
3. 精简提示词

### Q3：打包后的 EXE 无法运行

**可能原因**：
1. 缺少 VC++ 运行时库
2. 杀毒软件拦截

**解决方案**：
1. 下载安装 [VC++ 运行时库](https://aka.ms/vs/17/release/vc_redist.x64.exe)
2. 将 EXE 添加到杀毒软件白名单

### Q4：热键无效

**可能原因**：
1. 热键被其他程序占用
2. 程序未启动
3. 焦点在其他程序

**解决方案**：
1. 修改热键
2. 确认程序在运行（查看系统托盘）
3. 切换回本工具窗口

### Q5：截图标注不显示

**可能原因**：
1. 多显示器配置问题
2. DPI 缩放问题

**解决方案**：
1. 使用主显示器
2. 调整系统 DPI 设置为 100%

### Q6：变量追踪结果不准确

**可能原因**：
1. 源码语法错误
2. 变量名拼写错误
3. 动态生成代码无法分析

**解决方案**：
1. 先用 Python 语法检查工具检查源码
2. 确认变量名大小写正确
3. 对于动态代码，使用动态追踪器

### Q7：配置丢失

**可能原因**：
1. 配置文件被删除
2. 程序未正常退出

**解决方案**：
- 热键配置：重新设置热键
- AI 配置：重新填写 API Key 和模型信息
- 提示词配置：删除全部内容会自动恢复默认值

### Q8：API Key 安全吗？

**回答**：安全。

- API Key 存储在用户目录 `~/.gui_inspector/ai_config.json`，**不在项目目录中**
- **不会上传到 GitHub**（不在项目目录内）
- 日志中所有密钥均经过脱敏处理（如 `sk-x****6789`）
